  <p><br /></p><p><br /></p><p><br /></p><p>Testing multi-device interactions is now easier than ever with the Android Emulator. Whether you are building a multiplayer game, extending your mobile application across form factors, or launching virtual devices that require a device connection, the Android Emulator now natively supports these developer experiences.</p><p>Previously, interconnecting multiple Android Virtual Devices (AVDs) caused significant friction. It required manually managing complex port forwarding rules just to get two emulators to connect.</p><p>Now you can take advantage of a new networking stack for the Android Emulator which brings zero-configuration peer-to-peer connectivity across all your AVDs.</p><div><h3 style="text-align: left;">Interconnecting emulator instances</h3><div>The new networking stack for the Android Emulator transforms how emulators communicate. Previously, each virtual device operated on its own local area network (LAN), effectively isolating it from other AVDs. The new Wi-Fi network stack changes this by creating a shared virtual network backplane that bridges all running instances on the same host machine.</div></div>

  <h4>Key Benefits:</h4>
  <ul>
    <li><b>Zero-configuration: </b>No more manual port forwarding or scripting adb commands. AVDs on the same host appear on the same virtual network.</li><li><b>Peer-to-peer connectivity: </b>Critical protocols like Wi-Fi Direct and Network Service Discovery (NSD) work out of the box between emulators.</li><li><b>Improved stability: </b>Resolves long-standing stability issues, such as data loss and connection drops found in the legacy stack.</li><li><b>Cross-platform consistency: </b>Works the same across Windows, macOS and Linux.</li>
  </ul>

  <h3>Use Cases</h3>
  <p>
    The enhanced emulator networking supports a wide range of multi-device development scenarios:
  </p>
  <ul>
    <li><b>Multi-device apps: </b>Test file sharing, local multiplayer gaming, or control flows between a phone and another Android device.</li><li><b>Continuous Integration: </b>Create robust, automated multi-device test pipelines without flaky network scripts.</li><li><b>Android XR &amp; AI glasses: </b>Easily test companion app pairing and data streaming between a phone and glasses within Android Studio.</li><li><b>Automotive &amp; Wear OS: </b>Validate connectivity flows between a mobile device and a vehicle head unit or smartwatch.</li></ul><div><br /></div>

  <h3>Get Started</h3>
  <p>The new networking capability is enabled by default in the latest Android Emulator release (36.5), which is available via the <a href="https://developer.android.com/studio/intro/update#sdk-manager" target="_blank">Android Studio SDK Manager</a>.
